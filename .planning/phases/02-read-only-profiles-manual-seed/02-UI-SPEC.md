## Spacing Scale

8-point scale (all values are multiples of 4). No exceptions for Phase 2.

| Token | Value | Tailwind class | Usage in Phase 2 |
|-------|-------|----------------|------------------|
| xs | 4px | `p-1` / `gap-1` | Icon-to-text gap inside SourceBadge; freshness-dot-to-text gap |
| sm | 8px | `p-2` / `gap-2` | Chip inner padding; alias-row vertical rhythm; badge inner padding |
| md | 16px | `p-4` / `gap-4` | Default inter-element spacing inside a section; funding-row horizontal gaps |
| lg | 24px | `p-6` / `gap-6` | Between Hero sub-blocks; section inner padding on desktop |
| xl | 32px | `p-8` / `gap-8` | Between sections on desktop (Hero → Aliases → Rounds → Identifiers) |
| 2xl | 48px | `py-12` | Page top/bottom padding on desktop |
| 3xl | 64px | `py-16` | Not used in Phase 2 (reserved for future marketing pages) |

**Section rhythm contract:**
- Between sections: `32px` (desktop) / `24px` (mobile <640px)
- Within a section (heading → content): `16px`
- Between funding-round rows in table: `0` (table borders); between cards in mobile stack: `12px` via `gap-3` — **exception**: 12 is a multiple of 4, so this sits inside the scale as a half-step between `sm` and `md`. Allowed.

**Touch target minimum:** 44×44px for any interactive element (website link, future ♡ button slot). Satisfied by `h-11` on button variants.

## Typography

Exactly 4 sizes, exactly 2 weights.

| Role | Size | Weight | Line Height | Tailwind | Usage in Phase 2 |
|------|------|--------|-------------|----------|------------------|
| Display | 30px | 600 (semibold) | 1.2 (36px) | `text-3xl font-semibold leading-tight` | Hero `display_name_ko` only |
| Heading | 20px | 600 (semibold) | 1.3 (26px) | `text-xl font-semibold` | Section headings (별칭 / 투자 라운드 / 식별자); Hero `display_name_en` renders at this tier one step lower than display |
| Body | 14px | 400 (regular) | 1.5 (21px) | `text-sm font-normal leading-normal` | Default paragraph text, one-liner description, alias rows, funding-row cells, identifier values |
| Meta | 11px | 400 (regular) | 1.4 (≈15px) | `text-[11px] leading-tight` | SourceBadge pill text ("출처: DART · 2026-04-21 업데이트"), freshness labels, "비공개", "공시된 투자 라운드가 없습니다" inline empty |

**Rules:**
- **Never** mix weights other than 400 and 600 in a single view. Use size + color to create hierarchy, not a third weight.
- Hero `display_name_en` renders at Heading size (20px/600) with `text-muted-foreground` color — not at Body size — so the English name reads as "same-entity, secondary" rather than caption.
- Korean stage labels (`시리즈 A`, `Pre-A`, `SAFE`) render at Body size (14px) inside funding rows; the stage is not a heading.
- Badge uses non-scale 11px explicitly (per user Q2 confirmation). This is the ONLY place below 14px. Document as allowed exception.

**Color pairings:**
- Display / Heading → `text-foreground`
- Body (primary) → `text-foreground`
- Body (secondary, e.g. "HQ", "웹사이트" labels) → `text-muted-foreground`
- Meta (SourceBadge) → `text-muted-foreground`
- Stale-date Meta (when freshness dot is red) → `text-foreground` so the red dot is the only visual alarm, not the date color

## Color

60 / 30 / 10 split using existing shadcn `slate` tokens in `src/app/globals.css`. No new hex values introduced.

| Role | Token / Value | Usage in Phase 2 |
|------|---------------|------------------|
| Dominant (60%) | `--background` (oklch(1 0 0), near-white in light mode) | Page background, section backgrounds |
| Secondary (30%) | `--muted` (oklch(0.968 0.007 247.896)) + `--border` (oklch(0.929 0.013 255.508)) | Card surfaces (mobile funding cards), table row separators (`border-b`), SourceBadge pill background, alias section inner container |
| Accent (10%) | `--primary` (oklch(0.208 0.042 265.755) — near-black slate) | **Reserved for exactly 3 elements** (see list below) |
| Destructive | `--destructive` (oklch(0.577 0.245 27.325)) | **Not used in Phase 2** — no destructive actions in read-only profiles. Reserved for Phase 4b/4c. |

**Accent reserved for (exhaustive list — Q3 answer):**
1. **External website link** in Hero (the `website_url` anchor) — primary CTA of the page
2. **Lead-investor chip focus state** on funding-round rows — `ring` on keyboard focus and hover `bg-primary/10`; the lead chip itself renders with `font-semibold` border, accent fill only on interaction
3. **Current legal name emphasis** in the Aliases section — the active `valid_to IS NULL` alias row is rendered with `font-semibold` in `text-foreground`; no color fill, but this is the element that uses the accent "weight" of the page

Nothing else takes `--primary` as foreground/fill:
- Section headings use `text-foreground`, not accent
- Sector tag in Hero uses `--secondary` (gray chip), not accent
- Freshness dots use a **separate** semantic palette (see below), not the accent system

### Freshness semantic palette (separate from accent system)

Per D-02, thresholds are 30 / 180 days. Colors sit **outside** the 60/30/10 rule because they encode semantic trust signal, not brand.

| State | Threshold | Token / Tailwind | Hex (light mode) | Usage |
|-------|-----------|------------------|------------------|-------|
| Fresh | `≤ 30` days | `text-green-600` | `#16a34a` | Dot inside SourceBadge; "fact is recent" |
| Stale | `≤ 180` days | `text-amber-500` | `#f59e0b` | Dot inside SourceBadge; "verify before quoting" |
| Expired | `> 180` days | `text-red-600` | `#dc2626` | Dot inside SourceBadge; "data is aged" |

**Rules:**
- Dot size: `h-1.5 w-1.5` (6px), filled circle, rendered as `<span aria-hidden="true">` before "출처:" text inside the pill
- Screen-reader text: pill has visually-hidden span "{신선도 레이블}: {일수} 전 확인" (e.g., "신선: 12일 전 확인") so the color is not the only channel
- Colors are raw Tailwind, not tokenized — these three values live in `freshnessColor()` helper and nowhere else. No other component may use `green-600` / `amber-500` / `red-600` in Phase 2.
- Dark mode: `dark:text-green-500 / dark:text-amber-400 / dark:text-red-500` for WCAG contrast on `--card` in dark
- **Never** apply freshness color to body text; the dot alone carries the semantic. The date string stays `text-muted-foreground`.

| Hero section heading (visually hidden) | `profile.hero.srHeading` | 회사 개요 | Screen-reader-only so page has a landmark hierarchy |

| Funding rounds empty body | `profile.rounds.empty` | 공시된 투자 라운드가 없습니다 | D-Discretion-2 mandates this exact line for zero-rounds case |
| Funding amount undisclosed | `profile.rounds.undisclosed` | 비공개 | When `amount_minor IS NULL`. No freshness dot rendered alongside. |
| Identifiers section heading | `profile.identifiers.heading` | 식별자 | Visible `<h2>` |
| Identifiers empty body | `profile.identifiers.empty` | 등록된 식별자가 없습니다 | Rendered when no rows in `company_identifiers` |
| SourceBadge text | `profile.source.badge` | 출처: {sourceLabel} · {date} 업데이트 | ICU message. `sourceLabel` params: DART / 수동 큐레이션 / 사용자 제보. `date` pre-formatted with `date-fns` + Korean locale (e.g., "2026년 4월 21일"). |
| SourceBadge SR freshness labels | `profile.freshness.{fresh,stale,expired}` | 신선 / 확인 권장 / 오래된 정보 | Visually-hidden inside badge for screen readers |
| Source type labels | `profile.source.type.{dart,manual,user_submission}` | DART / 수동 큐레이션 / 사용자 제보 | Used by `sourceLabel()` helper |
| Error state (page-level, render-failure) | `profile.error.heading` + `profile.error.body` | **기업 정보를 불러오지 못했습니다** / 잠시 후 다시 시도해 주세요. 문제가 지속되면 `/sources` 페이지를 확인하세요. | Rendered in `error.tsx` for this route; body points researcher to trusted escape hatch |
| 404 state (slug not in DB) | `profile.notFound.heading` + `profile.notFound.body` | **요청하신 기업을 찾을 수 없습니다** / 검색 페이지에서 다른 기업을 찾아보세요. | Uses `notFound()` from next/navigation; body links to `/search` (Phase 3) — link text key `profile.notFound.searchCta` = "검색으로 돌아가기" |

**Destructive actions:** None in Phase 2. Explicitly confirmed by user (Q3: "destructive 미사용"). Do not introduce confirmation dialogs, toast-on-delete, or any `variant="destructive"` button in this phase.

**Voice rules:**
- Researcher tone (리서처 1순위). No marketing adjectives ("혁신적인", "최고의"). No exclamation marks.
- Action CTA copy uses verb + noun ("웹사이트 방문"), never "Click here" / "바로가기".
- Empty states describe the absence neutrally; never apologize ("죄송합니다" 금지).
- Error states point to a next action (검색으로 돌아가기 / 다시 시도 / `/sources` 확인).
- Freshness vocabulary is fixed: `신선 / 확인 권장 / 오래된 정보`. Do not introduce synonyms elsewhere.

## Responsive Contract

**Breakpoints:** Tailwind defaults; Phase 2 uses `sm` (640px) and `lg` (1024px) only. `375px` is the mobile floor per PROF-08.

| Viewport | Hero | Aliases | Funding rounds | Identifiers | Page padding |
|----------|------|---------|----------------|-------------|--------------|
| 375px (mobile floor) | logo 56px, display_name_ko wraps, one-liner clamped to 2 lines with inline expand, sector tag below | Vertical list, current bold first | Card-per-round (container query fires), labels repeated in each card | Vertical list, one row per identifier | `px-4 py-6` |
| 640px+ (sm) | logo 72px, names + CTA on same row | Same vertical list, wider container | `<table>` renders with 4 columns | Same vertical list | `px-6 py-8` |
| 1024px+ (lg) | logo 88px, Hero caps at `max-w-3xl` | Same | Table full width inside `max-w-4xl` container | Same | `px-8 py-12` |

**Container query rule:** Funding rounds section uses `@container` to switch between table and cards based on its own width, not the viewport. This keeps the table usable when the section later sits inside a narrower two-column layout (Phase 6+).

## Accessibility Contract

- All interactive elements have `focus-visible:ring-2 focus-visible:ring-ring focus-visible:ring-offset-2` per shadcn default
- Freshness color is paired with visually-hidden SR text (see Copywriting table)
- `<table>` includes `<caption class="sr-only">` = "투자 라운드 목록"
- `alt` text on company logo = `display_name_ko` (+ " 로고" suffix via i18n key `profile.hero.logoAltSuffix` = " 로고")
- `lang="ko"` inherited from route-level `/ko/*`; English names inside a Korean page wrapped in `<span lang="en">` when needed for screen-reader pronunciation
- Color contrast: all token pairings (`foreground` on `background`, `muted-foreground` on `background`, freshness colors on `muted`/`card`) pass WCAG AA ≥4.5:1 at Body size; 11px Meta text must meet ≥7:1 because of small size — the shadcn slate `muted-foreground` (oklch 0.554) on `background` (oklch 1.0) passes
- Keyboard navigation: entire page reachable with Tab; no keyboard trap; `Skip to main` link inherited from `(public)/layout.tsx`
