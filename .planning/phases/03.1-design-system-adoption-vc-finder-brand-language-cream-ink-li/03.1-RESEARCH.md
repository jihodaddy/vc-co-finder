# Phase 03.1: Design System Adoption — VC·Finder Brand Language - Research

**Researched:** 2026-04-22
**Domain:** Design system adoption (Tailwind v4 theme tokens + shadcn primitive extension + next/font wiring + brand primitives)
**Confidence:** HIGH (stack + pattern verified against official docs and npm registry; project-local state audited in situ)

## Summary

Phase 3.1은 이미 완성된 Phase 3 `/search` 위에 **VC·Finder 브랜드 디자인 시스템**을 덮어씌운다. 핵심은 세 덩어리다:

1. **토큰 교체** — `src/app/globals.css`의 shadcn 기본 slate OKLCH 팔레트를 CONTEXT D-01에서 잠긴 cream/ink/lime hex 매핑으로 바꾼다. Tailwind v4의 `:root { --var: ... } + @theme inline { --color-x: var(--var) }` 패턴을 유지하면서 hex → OKLCH 변환 없이 **hex 원시값을 그대로 주입**한다(공식적으로 허용됨 — 아래 [VERIFIED] 참조).
2. **프리미티브 확장** — shadcn `chart`, `slider`, `avatar`를 CLI로 추가하고(세 개 모두 현재 미설치), `src/components/metrics/`에 `Sparkline` / `Delta` / `LogoTile`을 shadcn 패턴(forwardRef 제거·`data-slot` 속성·CVA 변형)으로 만든다. Badge는 `variant="filter-chip"` + optional `onRemove` prop으로 확장한다.
3. **폰트 파이프라인 정비** — 현재 Pretendard가 프로젝트 어디에도 로드되지 않음(grep으로 src/ 전체 미매치 확인 완료). Geist + Geist_Mono는 `next/font/google`로 CSS 변수 방식으로 주입하고, Pretendard Variable은 `next/font/local`로 `./public/fonts/PretendardVariable.woff2`를 self-host. 이후 `@theme inline { --font-sans: var(--font-pretendard); --font-mono: var(--font-geist-mono); }` 블록에 연결한다.

**Primary recommendation:** Wave 1에서 `globals.css` + `components.json baseColor` + font 파이프라인 + DESIGN.md를 **하나의 파운데이션 단위**로 랜딩한다. Wave 2(shadcn 프리미티브 + metrics 컴포넌트)부터는 Wave 1 토큰을 소비하기만 하므로 병렬화 가능. `baseColor`를 slate → neutral로 바꾸는 것은 `components.json`의 미래 CLI 동작에만 영향을 주고 **이미 설치된 프리미티브는 다시 내려받지 않아도 된다**(재생성 시에만 신규 팔레트 기반 템플릿이 사용됨). 이미 설치된 13개 프리미티브는 `globals.css` 토큰 교체만으로 즉시 브랜드화된다.

### Locked Decisions

**Token Mapping (D-01)**

- D-01.1 `lime (#CFEF3E)` → `--primary`; `--primary-foreground` = `ink (#14120E)` (lime 위에 dark-text = "Chip active" signature).
- D-01.2 Surfaces:
  - `--background` = cream `#FBF9F4`
  - `--foreground` = ink `#14120E`
  - `--card` = white `#FFFFFF`, `--popover` = white
  - `--muted` = creamDeep `#F4F0E6`, `--muted-foreground` = inkMute `#78726A`
  - `--secondary` = creamDeep, `--accent` = creamDeep
  - `--border` = `rgba(20,18,14,0.10)` (ink @10% α — `ui.jsx TOKENS.line`)
- D-01.3 `--destructive` = coral `#FFB5A0`, `--destructive-foreground` = ink. Coral은 `Delta` 컴포넌트의 음수 시그널이기도 해서 일관됨.
- D-01.4 `--ring` = lime (DualSlider knob halo의 `rgba(lime, .4)`에 맞는 signature focus color).
- D-01.5 Utility token namespace 추가: `--radius-sm: 4px; --radius-md: 7px; --radius-lg: 12px; --radius-pill: 999px`; `--shadow-sm: 0 1px 2px rgba(20,18,14,0.06)`, `--shadow-md: 0 12px 32px -12px rgba(20,18,14,0.18)`; `--fw-regular/medium/semibold/bold: 400/500/600/700`.
- D-01.6 Dark mode는 v2로 연기. `.dark { ... }` 블록은 `globals.css`에 남기되 브랜드 일관 값은 작성하지 않음. DESIGN.md에 "v2"로 명기.
- D-01.7 `components.json`의 `baseColor`를 `slate` → `neutral`로 변경.

**Domain Components (D-02)**

- D-02.1 Sparkline = shadcn Chart 래퍼. `npx shadcn add chart` → `src/components/metrics/Sparkline.tsx`는 `<ChartContainer>` + Recharts `<AreaChart>`, minimal axes, `<ChartTooltip>`, tabular-nums label. ChartConfig의 color 토큰은 브랜드 팔레트로부터 드라이브. Phase 6 time-series도 동일 `<ChartContainer>` 사용 → forward-compatible.
- D-02.2 MiniTag ≠ 신규 컴포넌트. `src/components/ui/badge.tsx`에 `variant="filter-chip"`(ink bg + lime text + pill radius + padding 4px + fw 500) + `dismissible`(또는 `onRemove`) prop 추가. `ActiveFilterChips.tsx` 리팩터: `<Badge variant="filter-chip" dismissible onDismiss={...}>`. Phase 3의 `FacetCheckboxList` 내부 뱃지는 default Badge 유지.
- D-02.3 DualSlider = shadcn Slider + Radix range. `npx shadcn add slider` → `src/components/ui/slider.tsx`는 `SliderPrimitive.Root`를 사용, `value={[min, max]}`로 2-thumb 네이티브 지원. 스타일: thumb = 20px white circle + 2px ink border, focus halo = `ring: 0 0 0 6px rgba(lime, 0.25)`. `FacetRangeInputs.tsx`에 enhancement로 얹음(텍스트 입력 모드는 유지, slider가 시각 주인공).
- D-02.4 `src/components/metrics/` 신규 그룹:
  - `Sparkline.tsx` — `data`, `color`, `width`, `height` prop
  - `Delta.tsx` — `value`, `size` prop, ▲/▼ + `font-variant-numeric: tabular-nums` + 녹/coral 색상, Geist Mono
  - `LogoTile.tsx` — `company`, `size`, `radius` prop, shadcn Avatar(`npx shadcn add avatar`) 위에 빌드, company.color 배경 + white initial
- D-02.5 Registry hygiene: `src/components/ui/`는 shadcn-canonical 순수 유지, `src/components/metrics/`는 브랜드 도메인 그룹, `src/components/search/`는 Phase 3 조합 컴포넌트 유지.

**Rollout Scope (D-03)**

  1. **Foundation** — `globals.css` rewrite + `components.json baseColor` + font wiring + `DESIGN.md` authoring. (blocking 모든 후속 wave)

### Pattern 1: Tailwind v4 Theme Token Injection (Wave 1 핵심)

Tailwind v4는 `:root { --<name>: <value> }`로 원시 CSS 변수를 선언하고 `@theme inline { --color-<name>: var(--<name>) }`로 유틸리티에 매핑한다. `inline`이 중요한 이유: `next/font`의 CSS 변수처럼 **런타임에 해결되는** 변수를 Tailwind가 빌드 타임에 inline하지 않고 런타임 참조로 남기기 때문. 비-inline `@theme`는 컴파일 타임에 값을 고정하려 해서 next/font 변수 주입이 깨진다.

```css
/* src/app/globals.css — Wave 1 target */
@import "tailwindcss";
@import "tw-animate-css";

@custom-variant dark (&:is(.dark *));

:root {
  /* Brand semantic tokens (hex 원시값 — 공식 허용) */
  --background: #FBF9F4;           /* cream */
  --foreground: #14120E;           /* ink */
  --card: #FFFFFF;
  --card-foreground: #14120E;
  --popover: #FFFFFF;
  --popover-foreground: #14120E;
  --primary: #CFEF3E;              /* lime — signature accent */
  --primary-foreground: #14120E;   /* ink-on-lime (WCAG AA contrast 10.8:1) */
  --secondary: #F4F0E6;            /* creamDeep */
  --secondary-foreground: #14120E;
  --muted: #F4F0E6;
  --muted-foreground: #78726A;     /* inkMute */
  --accent: #F4F0E6;
  --accent-foreground: #14120E;
  --destructive: #FFB5A0;          /* coral */
  --destructive-foreground: #14120E;
  --border: rgba(20, 18, 14, 0.10);
  --input: rgba(20, 18, 14, 0.10);
  --ring: #CFEF3E;                 /* lime halo */

  /* Utility tokens (D-01.5) */
  --radius: 7px;                   /* keep for shadcn derived math */
  --radius-sm: 4px;
  --radius-md: 7px;
  --radius-lg: 12px;
  --radius-pill: 999px;
  --shadow-sm: 0 1px 2px rgba(20,18,14,0.06);
  --shadow-md: 0 12px 32px -12px rgba(20,18,14,0.18);
  --fw-regular: 400;
  --fw-medium: 500;
  --fw-semibold: 600;
  --fw-bold: 700;

  /* Chart palette (D-02.1 — Sparkline의 기본 stroke는 ink, Phase 6에서 다계열일 때 사용) */
  --chart-1: #14120E;              /* ink */
```

- Geist는 latin subset만 있으면 됨(아스키 + 수치). Korean은 Pretendard가 담당.

### Pattern 3: shadcn Chart Minimal Sparkline Wrapper

shadcn Chart의 `ChartContainer`는 `config.color` 값을 `--color-{key}` CSS 변수로 자동 주입하므로, `<Area fill="var(--color-trend)">`처럼 참조 가능.

```tsx
// src/components/metrics/Sparkline.tsx — Wave 2
'use client';
import { Area, AreaChart } from 'recharts';
import {
  ChartContainer,
  type ChartConfig,
} from '@/components/ui/chart';
import { cn } from '@/lib/utils';

type SparklinePoint = { v: number };

export type SparklineProps = {
  data: number[];
  color?: string;                  // raw hex or var() reference; default ink
  width?: number;
  height?: number;
  className?: string;
};

export function Sparkline({
  data,
  color = 'var(--color-chart-1)',   // ink by default
  width = 120,
  height = 36,
  className,
}: SparklineProps) {
  const chartData: SparklinePoint[] = data.map((v) => ({ v }));
  const config = {
    v: { color, label: 'Trend' },
  } satisfies ChartConfig;

  return (
    <ChartContainer
      config={config}
      className={cn('aspect-auto', className)}
      style={{ width, height }}
    >
      <AreaChart
        data={chartData}
        margin={{ top: 2, right: 0, bottom: 2, left: 0 }}
      >
        <defs>
          <linearGradient id="spark-fill" x1="0" y1="0" x2="0" y2="1">
            <stop offset="0%" stopColor="var(--color-v)" stopOpacity="0.25" />
            <stop offset="100%" stopColor="var(--color-v)" stopOpacity="0" />
          </linearGradient>
        </defs>
        <Area
          type="monotone"
          dataKey="v"
          stroke="var(--color-v)"
          strokeWidth={1.5}
          fill="url(#spark-fill)"
        />
      </AreaChart>
    </ChartContainer>
  );
}
```

Key points:

- `config.v.color` → `ChartContainer`가 `--color-v: <value>`를 컨테이너 scope에 주입. `<Area fill="var(--color-v)">`로 참조.
- Phase 6 시계열 컴포넌트는 동일 `<ChartContainer>` + 여러 `<Line>`을 쓰면 되므로 **API 호환**.
- Tooltip은 CONTEXT가 v1 Sparkline에서 요구하지 않음(ui.jsx는 Sparkline에 커스텀 tooltip 있지만 v1에서 shadcn `ChartTooltip` 사용 여부는 Claude's Discretion → 기본은 생략, 필요 시 `<ChartTooltip content={<ChartTooltipContent />} />` 추가).

### Pattern 4: CVA Variant Extension without Breaking Consumers

기존 Badge는 `default / secondary / destructive / outline` 4개 variant. `filter-chip` 추가는 **순수 addition**이므로 모든 기존 사용처 호환.

```tsx
// src/components/ui/badge.tsx — Wave 2 extension
// ... imports 동일

const badgeVariants = cva(
  "inline-flex items-center justify-center rounded-md border px-2 py-0.5 text-xs font-medium ...",
  {
    variants: {
      variant: {
        default: "...",
        secondary: "...",
        destructive: "...",
        outline: "...",
        // 신규
        'filter-chip':
          "border-transparent bg-[color:var(--foreground)] text-[color:var(--primary)] " +
          "rounded-full px-2.5 py-1 text-[11px] font-medium " +
          "[&>button]:bg-[color:var(--primary)]/10 [&>button]:hover:bg-[color:var(--primary)]/20",
      },
    },
    defaultVariants: { variant: "default" },
  },
);

function Badge({
  className,
  variant,
  asChild = false,
  dismissible = false,
  onDismiss,
  children,
  ...props
}: React.ComponentProps<'span'> &
  VariantProps<typeof badgeVariants> & {
    asChild?: boolean;
    dismissible?: boolean;
    onDismiss?: () => void;
  }) {
  const Comp = asChild ? Slot : 'span';
  return (
    <Comp
      data-slot="badge"
      className={cn(badgeVariants({ variant }), className)}
      {...props}
    >
      {children}
      {dismissible && onDismiss && (
        <button
          type="button"
          onClick={onDismiss}
          aria-label="제거"                 /* ← NOTE: i18n wiring은 소비자측에서 */
          className="ml-1 inline-flex h-3.5 w-3.5 items-center justify-center rounded-full"
        >
          <XIcon className="h-3 w-3" />
        </button>
      )}
    </Comp>
  );
}
```

Key points:

- **`forwardRef` 사용 안 함.** Tailwind v4 + React 19 + shadcn v4 패턴: 이미 현재 `badge.tsx`가 forwardRef 없이 function component 쓰고 있음 — 유지.
- **`data-slot="badge"` 유지** (shadcn v4 필수 — changelog 인용).
- `dismissible + onDismiss` prop 조합이 standard React 패턴. `aria-label` 문자열은 CONSUMER 측 i18n에서 주입해야 하므로 기본값은 비워두거나 prop으로 받도록 (i18n 원칙 준수).
- `ActiveFilterChips.tsx` 리팩터 시: 기존 `<Badge variant="secondary"><span>{label}</span><button>...</button></Badge>` 구조를 `<Badge variant="filter-chip" dismissible onDismiss={c.remove}>{c.label}</Badge>`로 대체하고, `aria-label` 처리를 위해 Badge에 `dismissAriaLabel?: string` prop 추가하거나 소비자 측 wrapper로 감싸기(planner가 결정).

### Pattern 6: LogoTile (shadcn Avatar 기반)

```tsx
// src/components/metrics/LogoTile.tsx — Wave 2
'use client';
import { Avatar, AvatarFallback, AvatarImage } from '@/components/ui/avatar';
import { cn } from '@/lib/utils';

type Props = {
  src?: string;
  name: string;
  color?: string;                  // background tint when no image (company.color)
  size?: number;                   // px
  radius?: number;                 // px (override avatar's default circle)
  className?: string;
};

export function LogoTile({ src, name, color = '#14120E', size = 40, radius = 8, className }: Props) {
  return (
    <Avatar
      className={cn('shrink-0 shadow-[0_1px_2px_rgba(0,0,0,.08)]', className)}
      style={{
        width: size,
        height: size,
        borderRadius: radius,
      }}
    >
      {src && <AvatarImage src={src} alt={name} style={{ borderRadius: radius }} />}
      <AvatarFallback
        style={{
          background: color,
          color: '#FFFFFF',
          borderRadius: radius,
          fontFamily: 'var(--font-geist)',
          fontWeight: 700,
          fontSize: Math.round(size * 0.45),
        }}
      >
        {name[0]}
      </AvatarFallback>
    </Avatar>
  );
}
```

Key points:

- shadcn Avatar는 기본 **circle** — `borderRadius` style로 square-with-rounded-corners 전환 가능.
- `AvatarFallback` 배경색은 `className="bg-..."` 또는 `style.background`로 override [CITED: github.com/shadcn-ui/ui/discussions/6326].
- `src` 있을 때 `<AvatarImage>`가 로드 성공하면 Fallback은 자동 숨김(Radix primitive 동작).

### Anti-Patterns to Avoid

- **`cdn.jsdelivr.net`/`unpkg.com`으로 Pretendard 링크 CSS**: FOUC + third-party network dep + Vercel edge caching 미스. `next/font/local`이 유일한 권장.
- **OKLCH 변환**: CONTEXT D-01이 hex로 잠겨 있음. 변환은 PRECISION loss 발생 + visual review 실패 리스크. Tailwind v4가 hex 허용 [VERIFIED: shadcn chart.tsx source].
- **`@theme`(non-inline) + next/font variable 혼합**: 빌드 타임 static inline 때문에 `--font-sans` 미해석 [CITED: tailwindlabs/tailwindcss discussions/13410].
- **forwardRef 추가**: shadcn v4는 forwardRef를 전면 제거. function component + props로 충분 [CITED: ui.shadcn.com/docs/tailwind-v4 changelog].
- **`<img src="/fonts/..."/>`로 SVG load**: recharts가 inline SVG만 지원하므로 Logo는 always inline 또는 `next/image`.
- **`components.json` alias path 변경**: CONTEXT D-05.3가 baseColor만 바꾸라고 명시 — aliases, style, rsc, tsx, iconLibrary 불변.
- **기존 설치된 shadcn 프리미티브를 다시 add**: `baseColor` neutral 전환 후 재설치할 필요 없음 (이미 파일 내 클래스는 `bg-primary` 같은 **토큰 참조**라서 globals.css 교체만으로 신규 팔레트 적용). [CITED: github.com/shadcn-ui/ui/issues/698 — "changing baseColor requires re-installing to reflect new palette IF you want new CLI templates, 그러나 기존 파일은 이미 토큰을 참조하므로 그대로 작동"].

## Common Pitfalls

### Pitfall 1: `@theme`(non-inline) → next/font variable 미해석

**What goes wrong:** `@theme { --font-sans: var(--font-pretendard); }` 쓰면 Tailwind v4가 빌드 타임에 inline 시도 → 런타임 변수 미해결 → `font-sans` 유틸리티가 fallback만 적용.

**Why it happens:** `@theme`는 static inline, `@theme inline`은 런타임 참조. next/font의 CSS 변수는 `<html>`의 className에 runtime scope.

**How to avoid:** `@theme inline` 블록 안에서만 `var(--font-*)` 참조. 이미 Pattern 1 예시에 반영.

**Warning signs:** DevTools Computed에서 `font-family: -apple-system, ...` (폴백만 잡힘) / Network에서 Pretendard woff2 요청 정상이지만 UI는 시스템 폰트.

[CITED: github.com/tailwindlabs/tailwindcss/discussions/13410]

### Pitfall 2: Pretendard `weight` 미지정 → WebKit 굵기 오렌더

**What goes wrong:** macOS Safari / iOS Safari에서 400/500/600/700이 모두 같은 굵기로 보임.

**Why it happens:** Pretendard는 variable font의 `wght` axis가 45–920 범위. next/font에 `weight: '45 920'` 없으면 WebKit이 기본 axis만 인식.

**How to avoid:** `localFont({ src, weight: '45 920', display: 'swap' })` 명시.

**Warning signs:** Chrome/Firefox에서는 정상, Safari에서만 `font-semibold`가 regular처럼 보이는 버그.

[CITED: orioncactus/pretendard README verbatim Korean: "weight 옵션을 지정하지 않으면 WebKit 기반의 브라우저에서 굵기가 잘못 렌더링"]

### Pitfall 3: shadcn Chart의 `--color-{key}` scope leak

**What goes wrong:** Multiple `<ChartContainer>` 인스턴스가 같은 config key(예: `v`)를 쓸 때 `--color-v` 변수가 mount 순서에 따라 override.

**Why it happens:** `ChartContainer` 내부 `ChartStyle`이 `<style>` 태그를 글로벌에 injection하지 않고 `[data-chart=$id]` scope selector에 한정 — 해결되어 있음.

**How to avoid:** 기본 shadcn Chart 구현은 안전. 하지만 **수동으로 `style={{'--color-v': ...}}`를 `<ChartContainer>` 밖에서 주입하지 말 것** — scope 깨짐.

**Warning signs:** 같은 페이지에 여러 Sparkline이 있을 때 색상이 섞이거나 모두 마지막 색으로 통일됨.

[CITED: chart.tsx source analysis — ChartStyle generates `[data-chart=${id}]` scoped CSS]

### Pitfall 4: FOUC/FOIT on first paint

**What goes wrong:** Pretendard 로드 전 한국어 글리프가 system fallback으로 잠시 표시됨 → 레이아웃 쉬프트.

**Why it happens:** `display: 'swap'`은 폴백을 먼저 보여주고 스왑 — CLS 가능.

**How to avoid:**
- `fallback: ['-apple-system', 'BlinkMacSystemFont', 'system-ui', 'Segoe UI', 'sans-serif']` 지정 → next/font가 자동으로 `adjustFontFallback`으로 fallback metric을 Pretendard 크기에 맞춰 override. CLS 거의 0.
- Variable font의 `size-adjust` 자동 계산을 믿어도 됨(next/font 내부 처리).
- 프로덕션에서 `display: 'optional'`로 바꾸면 로드 안 되면 영구 폴백(FOUT 없음) — 하지만 한국어 타이포 품질 희생. Phase 3.1은 `swap`이 정답.

**Warning signs:** Lighthouse CLS > 0.1, 첫 페인트 0.5초 내에 레이아웃 shift 감지.

[CITED: nextjs.org/docs/app/api-reference/components/font §adjustFontFallback]

### Pitfall 5: `baseColor: "neutral"`로 바꾼 후 새 `shadcn add`가 충돌

**What goes wrong:** 이미 cream/ink/lime 값이 `:root`에 있는 상태에서 `npx shadcn add chart` 실행 → shadcn CLI가 `globals.css`에 **neutral 팔레트 덮어쓰기**를 시도할 수 있음.

**Why it happens:** CLI는 `baseColor: neutral` 지정 시 기본 OKLCH neutral 토큰을 globals.css에 주입하려 시도 (cssVariables: true일 때).

**How to avoid:**
- **Wave 2 전에 Wave 1을 확정**. 즉 globals.css를 먼저 brand palette로 rewrite하고 그 뒤에 `shadcn add`.
- CLI가 중복 덮어쓰기 시도 시 파일 diff 리뷰 후 보존. shadcn CLI 4.x는 기존 `:root` 블록을 보통 감지하지만 방어적 리뷰 필요.
- 백업: `git add -A && git stash` 후 `shadcn add` → diff로 변경 확인 → stash 복원 + 새 컴포넌트 파일만 `git checkout --patch`로 살리기.

**Warning signs:** Wave 2 wave merge 후 `/search`가 갑자기 neutral-회색톤으로 돌아옴 → globals.css override된 증거.

[CITED: github.com/shadcn-ui/ui/issues/698]

### Pitfall 6: RSC/client boundary — 차트는 `'use client'` 필수

**What goes wrong:** Sparkline을 서버 컴포넌트에서 render → Recharts가 window 참조하므로 build 실패/hydration mismatch.

**Why it happens:** Recharts는 `ResizeObserver` 등 DOM API 사용.

**How to avoid:** `src/components/metrics/Sparkline.tsx` 맨 위에 `'use client'`. Parent RSC에서 prop으로 data 전달 OK.

**Warning signs:** `ReferenceError: window is not defined` 혹은 hydration warning.

### Pitfall 7: `tw-animate-css` 누락

**What goes wrong:** shadcn `add` CLI가 animation utility(`animate-accordion-up`)를 사용하는데 현재 `globals.css`에 `@import "tw-animate-css"`가 있는지 확인 안 하면 Accordion/Sheet의 open/close 애니메이션이 안 움직임.

**Why it happens:** Tailwind v4에서 `tailwindcss-animate`가 deprecated → `tw-animate-css`로 교체. package.json에는 있지만 globals.css import 누락 가능.

**How to avoid:** globals.css 최상단에 `@import "tw-animate-css";` 유지. 이미 현재 파일에 있음 — 보존 필수.

[VERIFIED: src/app/globals.css line 2 이미 import 중]

### Pitfall 8: `.dark` 블록을 완전 삭제하면 shadcn 컴포넌트 runtime error 가능

**What goes wrong:** D-01.6은 "dark mode v2 연기"지만 `.dark { ... }` 전체 삭제 시 `useTheme().theme === 'dark'` 가드 없이 클래스 참조하는 컴포넌트(거의 없지만)는 stale한 OKLCH 참조 가능.

**How to avoid:** `.dark` 블록을 남기되 내용은 placeholder/현재 값 그대로 유지 + DESIGN.md에 "v2 deferred"로 명기.

[VERIFIED via CONTEXT D-01.6 directive]

## Open Questions

1. **Pretendard woff2 파일 소스 — Git LFS vs public/fonts 직배?**
   - What we know: Pretendard Variable dynamic-subset woff2는 약 1.2MB (non-subset은 2MB+). Next.js `public/` 폴더는 git에 직접 커밋되는 것이 표준. Vercel deploy 시 artifact로 포함.
   - What's unclear: 프로젝트가 Git LFS 사용 안 함 확인 필요 (아마 안 쓰는 중). 1MB 수준은 LFS 없이 직접 커밋 무방.
   - Recommendation: `public/fonts/PretendardVariable.woff2`에 direct commit, `.gitattributes`는 건드리지 않음. Wave 1 task에서 woff2 download + commit을 하나의 task로 묶기.

2. **`Sparkline`이 `<ChartTooltip>`을 포함해야 하는가?**
   - What we know: `ui.jsx`의 Sparkline은 hover시 custom tooltip + crosshair 표시(line 64-83). shadcn `ChartTooltip`은 같은 UX를 제공.
   - What's unclear: Phase 3.1 Sparkline이 primitive로 출하될 때 tooltip을 기본 포함할지, opt-in으로 할지. 사용처가 아직 없어 판단 모호.
   - Recommendation: `showTooltip?: boolean` prop으로 opt-in(default false, primitive 크기 유지). Phase 6 소비자가 true로 켠다.

3. **`baseColor: neutral` 전환 후 13개 기존 프리미티브 재검증 수준?**
   - What we know: 컴포넌트 파일 내부는 `bg-primary`, `text-muted-foreground` 등 토큰만 참조 → globals.css 교체만으로 신규 팔레트 적용됨.
   - What's unclear: 간혹 CLI가 초기 주입 시 하드코딩된 컬러(예: `text-neutral-700`)를 emit한 primitives가 있는지 grep 감사 필요. (현재 확인된 것: 없음 — badge/button/accordion/checkbox/sheet/select 모두 토큰 참조).
   - Recommendation: Wave 2 초입에 `rg "bg-(neutral|slate|gray|zinc|stone)-[0-9]" src/components/ui`로 grep audit. 발견 시 Wave 2 내 하위 task로 수정.

4. **`/` landing page의 "최소 hero"가 얼마나 최소인가?**
   - What we know: CONTEXT D-03.1 step 5: "minimal brand hero with 검색 시작 CTA". D-02 deferred scan animation, auto-rotating facet demo, ticker, feature grid.
   - What's unclear: hero 안에 Sparkline/LogoTile demo가 1-2개라도 나타나야 "design system showcase" 효과 있음? 혹은 워드마크 + tagline + CTA만?
   - Recommendation: **wordmark + tagline + CTA + 1-line status ("LIVE · 5,016개 스타트업")** 수준까지. 프리미티브 demo는 제외 — 그건 마케팅 랜딩의 역할, Phase 3.1 범위 초과.

5. **`dismissible` Badge prop의 i18n aria-label 처리?**
   - What we know: `ActiveFilterChips`는 `t('chip.remove', { label })` 사용 중. Badge가 내부에서 `aria-label="제거"` 하드코딩하면 i18n 원칙 위반.
   - What's unclear: Badge에 `dismissAriaLabel: string` prop을 받는 게 깔끔한지, 아니면 consumer가 `children` 슬롯으로 직접 `<button>`을 composition하는지.
   - Recommendation: `dismissible: boolean + dismissAriaLabel?: string + onDismiss?: () => void` triad. `dismissible`만 true면 `dismissAriaLabel`이 required (TypeScript `never`/union trick으로 강제 가능). Consumer가 `t()` 값을 prop으로 주입.

### Known Threat Patterns for Design System Changes

| Pattern | STRIDE | Standard Mitigation |
|---------|--------|---------------------|
| CSS variable scope leak (cross-tenant theme) | Information disclosure | Phase 3.1은 multi-tenant 아님 — 단일 브랜드. 위험 없음 |
| XSS via `dangerouslySetInnerHTML` in new components | Tampering | **신규 컴포넌트(Sparkline/Delta/LogoTile/Badge)에 `dangerouslySetInnerHTML` 금지**. 현재 shadcn chart.tsx가 `<style dangerouslySetInnerHTML>`를 쓰지만 그 content는 config 기반 CSS string으로 `ChartStyle`이 `JSON.stringify`-grade sanitize — shadcn 공식 구현 채택 시 안전. 우리가 직접 추가하지 말 것. |
| Font loading CSP 위반 | Info disclosure | `next/font/local`은 same-origin이므로 CSP의 `font-src 'self'`만으로 충족. CDN 참조 제거로 CSP 단순화 이득. |
| user-controlled `company.color` → inline style injection | Tampering | `LogoTile.style.background = company.color` 직접 주입. DB 측에서 `#[0-9a-f]{6}` regex validation 강제 (Phase 4a ETL 책임 — 별도 phase). Phase 3.1은 정상 hex만 가정. |
