<decisions>
## Implementation Decisions

### Token Mapping (D-01)

- **D-01.1 — `lime (#CFEF3E)` → `--primary`**: shadcn Button default variant, Checkbox checked-state, Switch on-state, and focus rings all become lime automatically. `--primary-foreground` = `ink (#14120E)` because lime on light is low-contrast; dark-text-on-lime button is the brand signature ("Chip active" pattern in ui.jsx).
- **D-01.2 — Surfaces**: `--background` = cream (#FBF9F4), `--foreground` = ink (#14120E), `--card` = white (#FFFFFF), `--popover` = white, `--muted` = creamDeep (#F4F0E6), `--muted-foreground` = inkMute (#78726A), `--secondary` = creamDeep, `--accent` = creamDeep, `--border` = `rgba(20,18,14,0.10)` (ink at 10% α — `ui.jsx` `TOKENS.line`).
- **D-01.3 — Destructive stays distinct**: `--destructive` = coral (#FFB5A0) with `--destructive-foreground` = ink. Coral is the brand's "negative delta" signal in ui.jsx (`Delta` component when value < 0), so reusing it as destructive is consistent.
- **D-01.4 — Ring**: `--ring` = lime (signature focus halo matches ui.jsx DualSlider `rgba(lime).4` knob halo).
- **D-01.5 — Utility token namespace added**:
  - `--radius-sm: 4px; --radius-md: 7px; --radius-lg: 12px; --radius-pill: 999px` (derived from ui.jsx usage: MiniTag 4, button 7, card 12, Chip 999).
  - `--shadow-sm: 0 1px 2px rgba(20,18,14,0.06)`, `--shadow-md: 0 12px 32px -12px rgba(20,18,14,0.18)` (card hover elevation).
  - `--fw-regular: 400; --fw-medium: 500; --fw-semibold: 600; --fw-bold: 700`.
- **D-01.6 — Dark mode deferred**: keep `.dark` block in globals.css but do not author brand-consistent dark variables. Document in DESIGN.md as "v2".
- **D-01.7 — shadcn `components.json`**: update `baseColor` to `neutral` (closer to cream than `slate`) — affects any future `npx shadcn add ...` to emit neutral-based starter instead of blue-tinted slate.

### Domain Components (D-02)

- **D-02.1 — Sparkline via shadcn Chart**: run `npx shadcn add chart`. Build `src/components/metrics/Sparkline.tsx` as a thin wrapper around `<ChartContainer>` + Recharts `<AreaChart>` with minimal axes, hover `<ChartTooltip>`, and tabular-nums label. ChartConfig token colors drive from brand palette. Phase 6 time-series chart will use the same `<ChartContainer>` — API is forward-compatible.
- **D-02.2 — MiniTag = Badge variant**: do NOT create a separate `MiniTag` component. Extend `src/components/ui/badge.tsx`: add `variant="filter-chip"` (ink bg + lime text, pill radius, 4px padding, `font-weight: 500`) and a `dismissible` prop that renders an inline `<button>` with a cross glyph. Refactor `ActiveFilterChips.tsx` to render `<Badge variant="filter-chip" dismissible onDismiss={...}>`. Phase 3's `FacetCheckboxList` chips inside the sidebar stay with default Badge.
- **D-02.3 — DualSlider = shadcn Slider + Radix range**: run `npx shadcn add slider`. The resulting `src/components/ui/slider.tsx` uses `SliderPrimitive.Root` which supports `value={[min, max]}` and renders 2 thumbs natively with full keyboard accessibility. Style the track+range+thumb per ui.jsx DualSlider: thumb = 20px white circle with 2px ink border, focus halo = `ring: 0 0 0 6px rgba(lime, 0.25)`. Use in `FacetRangeInputs.tsx` as an enhancement (retain text-input mode for screen readers; slider is the visual primary).
- **D-02.4 — LogoTile, Delta — new `src/components/metrics/` group**:
  - `Sparkline.tsx` (wraps ChartContainer, props: `data`, `color`, `width`, `height`)
  - `Delta.tsx` (props: `value`, `size`) — ▲/▼ + `font-variant-numeric: tabular-nums` + color: green for positive, coral for negative; uses Geist Mono
  - `LogoTile.tsx` (props: `company`, `size`, `radius`) — builds on shadcn Avatar (`npx shadcn add avatar`) with initial fallback; company.color tints bg, white foreground text
- **D-02.5 — Registry hygiene**: `src/components/ui/` remains shadcn-canonical (no domain-mixed content). `src/components/metrics/` is the new brand-domain group for data-viz primitives. `src/components/search/` keeps existing Phase 3 composition components (uses ui + metrics).

### Rollout Scope (D-03)

- **D-03.1 — Wave-ordered rollout**:
  1. **Foundation**: `globals.css` rewrite + `components.json` baseColor + font wiring + `DESIGN.md` authoring. Blocking all other waves.
  2. **Primitives**: `npx shadcn add chart slider avatar`; new `metrics/` group; Badge variant extension.
  3. **Shell**: nav, footer, top-bar SearchInput to match ui.jsx shell. Global — must land before per-route styling.
  4. **`/search` rework**: ActiveFilterChips (→ Badge variant), result header live-count pulse, FacetSidebar typography (Geist Mono headers), FacetRangeInputs (new Slider), mobile "필터" pill polish.
  5. **`/` landing**: minimal hero page with "VC·Finder" brand mark, tagline, primary CTA → `/search`. Reuses new primitives.
  6. **`/companies/[slug]` cleanup**: freshness dot color, SourceBadge color, any remaining hardcoded hex scrubs. Tokens inherit automatically; this wave only fixes explicit violations.
- **D-03.2 — Routes NOT touched in Phase 3.1**: `/ko/login`, `/ko/sources`, `/ko/privacy`, `/ko/terms`. They inherit global tokens (cream bg, ink text, lime focus) automatically; no layout changes.
- **D-03.3 — Sparkline/Delta/LogoTile are built as primitives but not yet mounted on user-visible routes** in Phase 3.1. `LogoTile` can optionally land in `/search` result cards in wave 4 if bandwidth permits; otherwise all three are Phase 6 consumers.
- **D-03.4 — Test discipline**: each component added carries at least one Vitest unit test (render, core prop). `DESIGN.md` gets a visual-regression placeholder section noting "visual check via `npx next dev` screenshot, automated VRT is deferred to post-launch."

### Typography (D-04, Claude's Discretion)

- **D-04.1 — Pretendard Variable remains the body stack**: already wired in `src/app/[locale]/layout.tsx` / CSS. No change needed beyond confirming it renders across Windows/Mac.
- **D-04.2 — Geist + Geist Mono via `next/font/google`**: load both subsets via `import { Geist, Geist_Mono } from 'next/font/google'`. Expose as CSS variables `--font-geist`, `--font-mono-geist`. Geist Mono is the numeric + meta font.
- **D-04.3 — Usage contract**: `font-mono` class on all numeric cells (amounts, counts, metrics), meta labels (`LIVE`, `ACTIVE`, facet group headers), code-ish IDs; `tabular-nums` globally where Geist Mono is applied so columns align. Decided at planner level whether this is a global Tailwind utility class or per-component.

### Documentation (D-05)

- **D-05.1 — `DESIGN.md` is source of truth**: lives at `.planning/DESIGN.md` (project-level, not phase-local — future phases reference it). Contains:
  - Token catalog (all CSS vars with hex + usage note)
  - Typography scale + font stacks
  - Component inventory (shadcn primitives + metrics/* + any brand-extended variants) with props table
  - Usage rules (when to use `text-muted-foreground` vs raw hex; when Geist Mono vs Pretendard)
  - "v2 deferred" list (dark mode, full landing animation, Compare page visuals)
- **D-05.2 — `DESIGN.md` is authored during Wave 1** so subsequent waves cite it.
- **D-05.3 — `components.json`** stays in sync: baseColor update is the only change; alias paths unchanged.
