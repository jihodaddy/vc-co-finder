**Files:** `src/lib/data/freshness.ts`, `tests/unit/freshness.test.ts`

Replaced the three stock Tailwind color classes with brand-aligned arbitrary-value classes:

| Level | Before (Phase 2) | After (Phase 3.1 Wave 6) | Brand Rationale |
|---|---|---|---|
| `fresh` (≤30d) | `text-green-600 dark:text-green-500` | `text-[#0C8A3A]` | Delta-up green (DESIGN.md §1.2 chart palette, elevated from Delta-internal) |
| `stale` (≤180d) | `text-amber-500 dark:text-amber-400` | `text-[#B8860B]` | Dark goldenrod — warm-tone amber that does not compete with lime accent on cream |
| `expired` (>180d) | `text-red-600 dark:text-red-500` | `text-[#C03A3A]` | Delta-down coral-red (DESIGN.md §1.2, elevated from Delta-internal). The lighter `--destructive` (#FFB5A0) fails AA contrast on a 1.5px dot over cream, so the darker variant is used here |

Preserved:
- `FreshnessLevel` type shape (`'fresh' | 'stale' | 'expired'`)
- `freshnessLevel()` function signature + threshold constants (30 days, 180 days)
- The "single source of truth" guard comment (no component may hardcode the freshness palette)

Test updates:
- Replaced three `toMatch(/text-<color>-<shade>/)` assertions with exact `toBe('text-[#hex]')` assertions

Dark-mode variants (`dark:text-<color>-<shade>`) were intentionally dropped because DESIGN.md §5 v2 Deferred flags dark mode for v2; arbitrary-value classes also do not compose cleanly with `dark:` prefix for ad-hoc hexes.

## Deviations from Plan

### Scope-Alignment Decision (not a deviation — proactive guard)

**1. [Decision] Preserved existing FreshnessLevel type names**
- **Found during:** Task 1 initial read of `freshness.ts`
- **Context:** Plan draft used `'recent' | 'aging' | 'stale'` in prose and acceptance criteria; current code uses `'fresh' | 'stale' | 'expired'` (Phase 2 TRUST-05 API).
- **Decision:** Kept the existing level names. The plan's stated scope is a **color scrub**, not an API rename. Renaming levels would cascade to: SourceBadge sr-only i18n keys (`profile.freshness.fresh/stale/expired` → `profile.freshness.recent/aging/stale`), every SourceBadge consumer, and any telemetry/analytics keyed on the level string. Mapping honored exactly: the plan's `recent→#0C8A3A` becomes `fresh→#0C8A3A`, etc. Semantics are identical; only the hex values changed.
- **Tracked as:** No rule — this is a scope discipline decision documented in frontmatter `decisions`.
