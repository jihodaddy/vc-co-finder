Minimal single-screen brand hero at `/[locale]` — LIVE pill + wordmark + Geist headline + muted tagline + lime "검색 시작" CTA routing to `/[locale]/search`, with all copy flowing through a new `landing.hero3_1` next-intl namespace.

## Key Decisions

### D-1 — Non-destructive i18n namespace extension

Added `landing.hero3_1` as a **sub-object alongside** the existing `landing.hero` / `landing.valueProps` / `landing.status` keys rather than overwriting them. Rationale:

- Prevents breakage of any stale Phase 1 Plan 05 snapshot tests that may still reference `landing.hero.headline` / `landing.hero.ctaPrimary`.
- Keeps Wave 5 hero copy logically grouped for future audit (easy to grep `hero3_1` for "what shipped in 3.1").
- Trivial cost — an unused i18n sub-tree is ~200 bytes in the bundled message catalog, and the old keys document the Phase 1 intent for future contributors.

### D-2 — Hardcoded companyCount = 5016

The LIVE pill displays `{count, number}개 스타트업 · 한국·아시아`. Per CONTEXT D-03.2, this is the **first-touch brand surface** — a live company count reinforces the "live · multi-facet" value prop. However:

- Phase 3 has no public `getCount()` Server Action yet.
- Wiring one in Wave 5 would leak scope (requires DB read, caching strategy, revalidation plan).
- 5016 matches the Phase 2 SRCH-13 cold-start baseline + Phase 8 LAUNCH-03 projection for v1 shipped seed size.

The page comment flags this for a Phase 8 data-wiring task. See **Known Stubs** below.

### D-3 — Reuse Wave 3 + Wave 4 patterns for cross-route brand continuity

Rather than authoring a landing-specific wordmark or LIVE pill, the page composes two existing Wave patterns inline:

- **Wordmark**: lime `h-3 w-3 rounded-[3px]` square + Geist bold `VC·Finder`. Matches `src/components/site/header.tsx` wordmark at a larger scale.
- **LIVE pulse**: `animate-ping` + `bg-[color:var(--primary)]` dot. Matches Wave 4 `src/components/search/ResultsHeader.tsx` pulse dot.

Benefit: navigating between `/` → `/ko/search` shows the **same pulse + lockup language** in different contexts. No new primitives introduced.

### D-4 — Deferred content (CONTEXT D-03.2 lock-in)

The following are **explicitly absent** from this wave and enforced by the regression guard test (`<article>` count === 0, `<section>` count ≤ 2):

- 3-up value-prop `<article>` grid (Phase 1 layout).
- Full `landing.jsx` marketing scan animation (`scanPos` gradient sweep).
- Auto-rotating facet chip demo (2.2s interval `setActiveFacet`).
- Companies ticker strip (35s `animation: scroll` LogoTile row).
- Feature grid (3 big blocks: Faceted / MetricTrends / Side-by-side).
- Users section + footer CTA.

All tracked under `.planning/DESIGN.md §5 v2 Deferred` as "full landing marketing experience" — a post-launch marketing-phase decision.
