# Phase 03.1 Plan 04: /search Wave 4 Rework — Badge filter-chip + Slider + Geist Mono headers + LIVE pulse + top-bar shadow

**One-liner:** Refactor `/search` surface to consume Wave 2 brand primitives — `ActiveFilterChips` uses the `Badge filter-chip dismissible` API, `FacetRangeInputs` gains a shadcn dual-thumb `<Slider />` alongside retained text inputs, facet headers and the new `실시간` label render in Geist Mono meta-label style, `ResultsHeader` pulses a lime `animate-ping` dot per ui.jsx LIVE pattern, and `SearchInput` adopts the landing.jsx hero top-bar shadow + `rounded-[14px]` container — all with zero change to existing functional behavior (nuqs URL state, autocomplete, sort, pagination, parseKRW, aria-invalid).

### Task 2 — FacetRangeInputs + shadcn Slider overlay (commit `554f8b4`)

Added a dual-thumb `<Slider />` above the existing text input row:

```tsx
<Slider
  value={[sliderMin, sliderMax]}
  min={domainMin}
  max={domainMax}
  step={step}
  onValueChange={([mn, mx]) => { setMinTxt(String(mn)); setMaxTxt(String(mx)); }}
  onValueCommit={() => commit()}
  aria-label={t(`facet.${paramKey}.label`)}
  className="py-2"
/>
<div className="flex gap-2">
  <Input ... onBlur={commit} onKeyDown={... commit() on Enter} />
  <Input ... onBlur={commit} onKeyDown={... commit() on Enter} />
</div>
```

- `sliderMin` / `sliderMax` are **derived state** computed from `minTxt` / `maxTxt` via `parseKRW` (funding branch) or `parseInt` (founded branch); empty text snaps to domain min/max.
- Domain constants: `MAX_FUNDING = 1_000_000_000_000` (1조 amount_minor), `MIN_YEAR = 1970`, `MAX_YEAR = new Date().getFullYear()`, `step = funding ? 100_000_000 : 1`.
- **Preserved:** `commit()`, `parseRange()`, `paramKey` branching, `aria-invalid` for `min>max`, two `<Input onBlur={commit}>` elements, page-reset-to-1-on-commit behavior.
- **Test file updated:** `tests/unit/facet-range.test.tsx` — added 2 new assertions: (1) `screen.getAllByRole('slider').length >= 2` verifies dual-thumb rendering; (2) founded-paramKey slider exposes `aria-valuemin=1970` and `aria-valuemax=currentYear` on its max thumb. All pre-existing assertions on Enter/blur commit, invalid min>max state, and `parseKRW` funding-branch behavior continue to pass because the text Input elements are **unchanged**.

## Slider Domain Rationale

- **Funding: 0 → 1조 (1,000,000,000,000 amount_minor), step 1억:** Phase 2 seed corpus tops out near 5,000억; 1조 gives headroom for Phase 4+ ETL additions. Step = 1억 (100,000,000) yields a dual-thumb resolution of 10,000 discrete positions — draggable without jitter, precise enough for "100억 ~ 1,000억" typical filter ranges.
- **Founded: 1970 → currentYear, step 1:** 1970 is the earliest `founded_at` in the Phase 2 seed fixtures (and predates most real Korean startup incorporation by a safe margin). Using `new Date().getFullYear()` auto-advances the upper bound; no maintenance required on January 1.
- **Why a 2-element domain array and not a shared constant:** The two facets have fundamentally different numeric shapes (12-digit won amount vs 4-digit year) — a shared domain would be meaningless. Local constants in `FacetRangeInputs.tsx` keep the mapping readable.
