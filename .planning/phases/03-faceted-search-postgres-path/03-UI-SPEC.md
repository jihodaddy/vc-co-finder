## Spacing Scale

Inherits Phase 2 `02-UI-SPEC.md §Spacing Scale` verbatim. No new tokens. **Phase 3 usage overlay** below.

| Token | Value | Tailwind | Phase 3 usage |
|-------|-------|----------|---------------|
| xs | 4px | `p-1` / `gap-1` | Chip icon-to-text gap (`X` close glyph beside chip label); checkbox label-to-count gap |
| sm | 8px | `p-2` / `gap-2` | Chip row inner gap; facet sidebar item vertical rhythm; table cell vertical padding |
| md | 16px | `p-4` / `gap-4` | Facet group inner padding; table cell horizontal padding; card grid inner padding; search-input padding |
| lg | 24px | `p-6` / `gap-6` | Between results-area header (chips + count + sort + view toggle) and results body; inside desktop sidebar between facet groups |
| xl | 32px | `p-8` / `gap-8` | Between main page title area and sidebar-plus-results grid (desktop) |
| 2xl | 48px | `py-12` | Page top/bottom padding on desktop |

**Phase 3 layout spacing contract:**

- Desktop (≥1024px, `lg`): two-column grid `grid-cols-[minmax(280px,320px)_1fr] gap-8`. Sidebar is **fixed-width** (not fluid): `280px` min, `320px` max. `sticky top-16` inside the sidebar container so the filter panel stays visible while results scroll.
- Tablet (`sm` 640–1023px): single column, sidebar collapses into a top "필터" trigger button; results render below chips.
- Mobile (<640px): single column, `px-4 py-6`. "필터" trigger is fixed to bottom of viewport as a pill button (`bottom-4 right-4`, `shadow-lg`) that opens the bottom sheet drawer.
- Chip row to results body: `gap-6` (24px).
- Between table rows: `0` (table borders); between cards in card-grid view: `gap-4` (16px) horizontally, `gap-4` vertically.

**Touch target minimum (carry-forward):** `h-11` (44px) on every tap surface: "필터" button, drawer "적용" button, pagination arrows, sort buttons in card view (on mobile). Checkbox rows in the facet drawer must render at `h-11` on mobile even though desktop can drop to `h-8`.

Exceptions: none beyond the 11px Meta text exception already documented in Phase 2.

## Typography

Inherits Phase 2 `02-UI-SPEC.md §Typography` verbatim. 4 sizes, 2 weights. **Phase 3 assignment table** below.

| Role | Size / Weight / LH | Tailwind | Phase 3 usage |
|------|---------------------|----------|---------------|
| Display (30px / 600 / 1.2) | `text-3xl font-semibold leading-tight` | **Not used on `/search`** — the page has no hero; page title is visually-hidden `<h1 class="sr-only">검색</h1>` for a11y landmark only |
| Heading (20px / 600 / 1.3) | `text-xl font-semibold` | Facet-group labels in sidebar ("섹터", "라운드 단계", ...); "0개 기업" empty-state heading |
| Body (14px / 400 / 1.5) | `text-sm font-normal leading-normal` | Table cell text (company name, sector, stage, funding amount, employee count, founded year); facet item labels; active-chip labels; sort-by label; search-input text; autocomplete suggestion rows |
| Meta (11px / 400 / 1.4) | `text-[11px] leading-tight` | Facet item count suffix ("(127)"), table column-header labels (uppercase? NO — Phase 2 rule "no third weight" + no case transforms; rendered at Meta size for visual secondarization), pagination "X-Y of Z개 기업" summary, "마지막 업데이트" annotations in result rows |

**Weight rules (carry-forward):** Only `400` and `600`. Active state on sort columns uses Body size + `font-semibold`, not a third weight.

**Result-count typography rule:** The live result count ("1,247개 기업") renders at **Heading size** (20px / 600), not Display — the count is an information-first metric, not a brand moment.

**Active-chip typography:** 14px Body (`text-sm`), regular weight, no uppercase. Chip is distinguished by shape + border color, not by font.

**Emphasis inside table rows:** Company name column uses `font-semibold` + `text-foreground`; every other column is `font-normal` + `text-muted-foreground` for non-primary metadata (stage label, latest-round date) and `text-foreground` for quantitative values (cumulative funding, employees). This keeps the name as the scan anchor.

## Color

60 / 30 / 10 split using existing shadcn `slate` tokens in `src/app/globals.css`. **No new hex values introduced.** Inherits Phase 2 `02-UI-SPEC.md §Color` entirely. Phase 3 accent reserved-for list extends the Phase 2 list without replacing it.

| Role | Token | Phase 3 usage |
|------|-------|---------------|
| Dominant (60%) | `--background` | Page background, results-table background, card background at rest |
| Secondary (30%) | `--muted` + `--border` + `--secondary` | **Facet sidebar surface** (`bg-muted/40` or `bg-card` with `border-r`), active-chip background (`bg-secondary`), table row hover (`bg-muted/40`), drawer surface, autocomplete popover background |
| Accent (10%) | `--primary` | **Reserved-for list below** |
| Destructive | `--destructive` | **Not used in Phase 3.** No destructive actions on `/search`. A filter removal is not destructive — it's a UI reset, rendered in the Secondary palette. |

**Accent reserved for (Phase 3 extension of Phase 2 list — exhaustive):**

1. **Active-state sort column header** (desktop table) — the column currently sorting the result set renders its sort-arrow glyph (`ArrowUp` / `ArrowDown`) in `text-primary`. Column label remains `text-foreground`; only the glyph carries accent. All other sort glyphs use `text-muted-foreground`.
2. **Active-view-toggle segment** — the currently-selected view (table or card) in the Table/Card toggle uses `bg-primary text-primary-foreground`. The inactive segment is `bg-transparent text-muted-foreground`.
3. **Focus ring on every interactive element** — `ring-2 ring-ring ring-offset-2 ring-offset-background` on `focus-visible` (shadcn default; `ring` variable is slate, not primary — this is a11y, not accent — kept out of the "10%" count).
4. **Active checkbox fill** (facet items) — shadcn Checkbox default renders `bg-primary` when `checked`. This is expected accent usage and does not require a custom variant.
5. **Current-page number** in numeric pagination — the active page number has `bg-primary text-primary-foreground`; other page numbers are `bg-transparent hover:bg-muted`.
6. **Autocomplete keyboard-highlighted suggestion** — while `ArrowDown`/`ArrowUp` navigates the suggestion list, the focused suggestion renders `bg-primary/10 text-foreground` (NOT full primary fill — this is a navigation affordance, not a final selection). On `Enter` or click, selection commits.

**Nothing else takes accent:**
- Facet-group headings: `text-foreground`.
- Active-filter chip label: `text-foreground`; chip border: `border-border`; chip background: `bg-secondary`.
- Table row hover: `bg-muted/40`, no accent.
- "모두 지우기" link: `text-muted-foreground underline-offset-4 hover:text-foreground hover:underline` — deliberately de-emphasized, not a primary CTA.
- Result count "1,247개 기업": `text-foreground`, no accent. (This is live data display, not a callout.)

### Freshness semantic palette (Phase 2 carry-forward — usage in Phase 3)

**Per CONTEXT `Phase 2 carry-forward`** block: `freshnessLevel` + `FRESHNESS_DOT_CLASS` from `src/lib/data/freshness.ts` are available. **In Phase 3 results rows, we do NOT render a per-row SourceBadge** (CONTEXT deferred explicitly: "리스팅 밀도 우선, per-row badge는 `/companies/[slug]`만").

**Phase 3 freshness rule:** On **both table and card views**, render a **bare freshness dot only** (no accompanying "출처:" pill, no date text) at the right edge of the "최신 라운드" column (table) or under the latest-round row in card view, with:

- `h-1.5 w-1.5` (6px) dot using `FRESHNESS_DOT_CLASS[level]` classes (reuses `text-green-600 / text-amber-500 / text-red-600` + dark-mode counterparts already tokenized in Phase 2)
- `aria-hidden="true"` on the dot itself
- Visually-hidden accompanying `<span class="sr-only">{freshness label} · {formatProfileDate(lastVerifiedAt)}</span>` so screen-readers still get the semantic
- **Threshold source of truth:** `src/components/profile/SourceBadge.tsx` — Phase 3 must NOT define a second palette. If `freshnessLevel` ever needs to change, it changes in one place.

Rationale: researcher scans a 25-row table; three columns of "출처: 수동 · 2026-04-22" badges create noise. The dot alone carries the semantic signal at list density; deep provenance remains one click away on `/companies/[slug]`.

**Destructive actions:** None. "모두 지우기" is a URL reset — no confirmation dialog, no `variant="destructive"`. Removing a chip / unchecking a facet is immediate.

**Voice rules (carry-forward + delta):**
- Researcher tone (1순위). No marketing adjectives. No exclamation marks.
- Empty state describes absence + offers 1 specific next step ("필터를 완화하거나 모두 지우기").
- Error state points to next action ("다시 시도" + `/sources`).
- Freshness vocabulary fixed: `신선 / 확인 권장 / 오래된 정보` (Phase 2 carry-forward; used in sr-only freshness dots).
- **Never** "결과가 없어요" (casual) — always "0개 기업" (factual) + neutral body.
- **Never** "죄송합니다" — error state is neutral direction.
- Stage labels render via Phase 2 `stageLabel()` helper; do NOT introduce new stage copy.

### D-01 Desktop sidebar contract

- Width: `min-w-[280px] max-w-[320px] w-[18rem]` (`w-72 = 288px`, chosen within range).
- All 6 facet groups are **always expanded** (no accordion, no collapse). Group heading is a plain `<h2>` at Heading size. Item list follows immediately.
- **Show more/less:** Facet groups with > 8 items (sector, region) render the first 8 items + a `<button>` "더 보기 ({remaining}개)" at Body size with `text-muted-foreground hover:text-foreground`. Clicking reveals the rest inline (no animation) and swaps the button label to "접기".
- **Facet item row:** `<label class="flex items-center gap-2 h-8 cursor-pointer">` + `<Checkbox>` + facet label (14px) + count suffix `(127)` in Meta size + `text-muted-foreground`.
- **Facet group order (D-04 locked):** sector → stage → region → employees → funding → founded.
- **Separator between facet groups:** `<Separator class="my-4" />` — visual group boundary without accordion boilerplate.
- **Search input** lives at the **top of the sidebar** (above the first facet group), inside a small `<div class="pb-4 border-b mb-4">` container so it reads as a search affordance, not another facet.

### D-02 Mobile drawer contract

- Trigger: fixed pill button `bottom-4 right-4`, `h-12 px-6 rounded-full shadow-lg bg-primary text-primary-foreground`. Label: "필터" or "필터 ({count})" if activeCount>0.

- Content: `<Accordion type="multiple" defaultValue={['sector']}>` — 6 items, sector expanded by default, others collapsed. `<AccordionTrigger>` renders facet label at Heading size with count `(3)` suffix in Meta size when that facet has active selections.
- Footer: sticky "적용" button at `h-11`, `w-full`, `variant="default"`. Apply closes the sheet (state is already committed to URL as each checkbox toggled).
- Close affordance: top-right `X` icon button (`h-10 w-10`), sr-only label "닫기", in addition to swipe-down-to-dismiss inherited from shadcn Sheet.

### D-03 Active-filter chip bar contract

- Position: **always above `<ResultsHeader />` and `<ResultsTable />`**. Renders even on mobile (below the drawer trigger / search input).
- When `activeFilterCount === 0`: renders `null` (no empty row).
- When `activeFilterCount > 0`: `<ul class="flex flex-wrap gap-2 items-center">` of chip `<li>` elements + trailing `<button>` "모두 지우기".
- **Chip shape:** `<Badge variant="secondary">` with internal layout `inline-flex items-center gap-1` + facet-value label + `<button aria-label="{label} 제거" class="inline-flex h-4 w-4 items-center justify-center rounded-full hover:bg-muted-foreground/20"><X class="h-3 w-3" /></button>`.
- **Chip label format per facet type:**
  - Sector / stage / region (multi): one chip per selected value, e.g., "핀테크", "시리즈 A", "서울".
  - Employees (bucket): "직원 51-200명".
  - Funding (range): "투자 100억원 이상" when only `min` set; "투자 10-100억원" when both set; `formatKRW` for amount.
  - Founded (range): "설립 2020년 이후" / "2020-2024년" / "2020년 이전".
- **"모두 지우기":** inline at end of chip row, `text-sm text-muted-foreground underline-offset-4 hover:text-foreground hover:underline`. Not a button variant.
- **Accessibility:** Chip bar wrapper has `role="group" aria-label="활성 필터"`; each chip remove button announces `{facet label} 제거`.
- **Interaction:** Clicking chip X removes just that value; clicking "모두 지우기" resets all facets **except** view/sort/page (those persist).

### Autocomplete (discretion resolved)

**Scope:** **Single global search input** placed at the top of the desktop sidebar (above the first facet group) and at the top of the results column on mobile. Queries `aliases` + `companies` tables via `lib/search/adapter.ts`; resolves 토스 ⇄ 비바리퍼블리카 ⇄ Toss to the same canonical `company_id`.

**Interaction:**
- Trigger: `<input>` wrapped in `<Popover>`. Typing with length ≥ 1 opens the popover.
- Popover content: `<Command shouldFilter={false}>` (server-driven list, client filter disabled).
- Suggestion row: `<CommandItem>` = `<CompanyLogo h-5 w-5 />` + display_name_ko (font-semibold) + " " + alias hint `<span class="text-muted-foreground text-xs">({aliasType})</span>` when query matched an alias, not the display name. Example suggestions for "토스": `토스 (브랜드)`, `비바리퍼블리카 (법인명)`, `Toss (영문명)`.
- Keyboard: `ArrowDown`/`ArrowUp` navigate, `Enter` selects (navigates to `/companies/{slug}`), `Escape` closes.
- Debounce: 150ms on input change before firing suggestion query.
- Empty result: `<CommandEmpty>` renders `search.autocomplete.empty` copy.
- Loading: visible `search.autocomplete.loading` row after 120ms.
- **Autocomplete selection commits navigation**, not filter state — selecting a suggestion opens the profile page. The search input is a jump-to-company affordance, not a free-text facet. (Free-text filtering is a v2 concern per SRCH-V2-01.)

**Non-goals (explicitly):**
- No autocomplete suggestions for sector / stage / region values. Those are facet checkboxes only.
- No "recent searches" history. Sessionless page.

### Range facet UX (discretion resolved)

**Employees (D-04 #4):** **Bucket chips** + "직접 입력" popover escape hatch.

- Bucket chips (single-select; semantically a radio group): `1-10명` / `11-50명` / `51-200명` / `201-500명` / `501-1,000명` / `1,000명 이상` — industry-standard range (matches LinkedIn, Crunchbase).
- Chips render as `<button variant="outline" class="h-8 px-3 rounded-full">`. Selected chip: `bg-primary text-primary-foreground border-primary`.
- "직접 입력" trailing button opens a `<Popover>` with two `<Input type="number" inputMode="numeric">` (최소 / 최대) and an "적용" button. Applying commits a custom range that does not match a bucket; URL reflects as `employees=75-225`.
- Only ONE active state: either a bucket OR a custom range. Selecting a bucket clears the custom range and vice versa.

**Cumulative funding (D-04 #5):** **From/to numeric inputs** with unit hint "원" + Korean placeholder like "100억" using `formatKRW`-aware parser.

- Two `<Input type="text" inputMode="decimal">` side by side, labels "최소" / "최대", with small hint text below: "100억" and "1조" as placeholders.
- Parser accepts "100억", "1.5조", "500000000", "5억원" — normalizes to `amount_minor` BIGINT via a new `parseKRW` helper (lives in `src/lib/format/currency.ts` or `src/lib/format/parseKRW.ts`).
- URL encoding: `funding=10000000000-100000000000` (raw `amount_minor` values, no units). The chip bar formats back to "10억-1,000억원" via `formatKRW` for human display.
- Validation: if `min > max`, render input in `aria-invalid="true"` with border `--destructive`; chip does not commit.

**Founded year (D-04 #6):** **From/to numeric inputs**, 4-digit year.

- Two `<Input type="number" inputMode="numeric" min={1900} max={currentYear}>`, labels "최소 연도" / "최대 연도".
- URL: `founded=2020-2024`.
- No buckets — year is a compact scalar; input is faster than chip scan.

**Rationale for rejecting dual-sliders:** sliders introduce gesture imprecision at Korean numeric scales (1억 vs 10억 matters); numeric inputs let researchers commit exact values. Sliders were the obvious default but researcher tone favors precision over polish.

### Loading state (discretion resolved)

- **Results area:** `<ResultsSkeleton />` renders **inline in the results `<Suspense>` boundary**, not as a full-page spinner.
- **Shape parity with the final view:** if `view=table`, skeleton is a table with 10 skeleton rows (each with 6 `<Skeleton class="h-4 w-[X%]">` cells matching column widths). If `view=card`, skeleton is a grid of 10 skeleton cards.
- **Duration target:** with Postgres + GIN on seed data, p95 <1s (ROADMAP SC #4); skeleton should usually flash only briefly. Still required — fallback hides empty flicker when the query takes >200ms.
- **`aria-busy="true"`** on the results container during skeleton render; SR announcement `search.loading.srLabel` = "검색 결과를 불러오는 중".
- **Chip bar + header do NOT skeleton.** They render with the known URL state instantly; only the result rows are pending.
- **Count while loading:** the `{count}개 기업` number falls back to a `<Skeleton class="h-7 w-24" />` of matching size until the count arrives.

### URL state encoding (discretion resolved — nuqs contract)

**nuqs serializers:**
- Multi-select facets (sector / stage / region): `parseAsArrayOf(parseAsString)` with **comma-separated** output (CONTEXT §Claude's Discretion + ROADMAP SC #1 example). E.g., `?sectors=fintech,ai`.
- Range facets (employees / funding / founded): `parseAsString` with `-` separator (e.g., `employees=50-500`). Open-ended ranges use empty side: `employees=50-` or `funding=-10000000000`.
- Bucket (employees bucket): `parseAsStringLiteral(['1_10','11_50','51_200','201_500','501_1000','1001_plus'])`. Custom ranges override bucket (validator ensures exclusivity).
- View: `parseAsStringLiteral(['table','card']).withDefault('table')`.
- Sort: `parseAsStringLiteral([...8 values]).withDefault('recent_funding_desc')`.
- Page: `parseAsInteger.withDefault(1)`.
- Per-page: `parseAsStringLiteral(['25','50','100']).withDefault('25')`.
- Autocomplete query: **NOT persisted** — transient client state only.

**Shareability test (SRCH-06):** Opening `?sectors=fintech,ai&stage=series_a,series_b&region=KR&employees=50-500&sort=recent_funding_desc&view=table&page=1&per_page=25` in a new tab must reproduce an identical UI state. This is a Phase 3 validation requirement already locked by ROADMAP SC #2.

## Interaction Timing & Motion

- **Nuqs URL commit on filter toggle:** synchronous (debounce 0). Each checkbox toggle updates URL immediately; results query debounces at the data layer (300ms) to coalesce rapid toggles.
- **Debounce policy:**
  - Search autocomplete: 150ms from last keystroke before fetch.
  - Range from/to input: commit to URL on blur OR Enter key; typing does NOT commit (prevents partial-range flicker).
  - Live result-count announcement (sr-only): 300ms after last URL change.
- **Skeleton minimum display time:** 0ms (no forced "smooth flash"); skeleton appears only when the query is still pending.
- **Drawer animation:** shadcn Sheet default (slide-up from bottom, 300ms ease-out). No custom keyframes.
- **Popover animation:** shadcn defaults via `tw-animate-css`.
- **No layout animation** on chip add/remove (prevents jank). Chips appear/disappear with `tw-animate-css` `fade-in-0 fade-out-0` (ms) but no position tween.
- **No transitions on sort-arrow swap** (`text-muted-foreground` → `text-primary`): instant. Researchers should see the state change as a discrete commit, not a fade.
