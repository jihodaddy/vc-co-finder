1. `src/app/auth/callback/route.ts` — Supabase OAuth callback handler. Exchanges the `code` query param for a session, then redirects to `next` (default `/ko/`):
```ts
import { NextRequest, NextResponse } from 'next/server';
import { createClient } from '@/lib/supabase/server';

/**
 * OAuth callback per https://supabase.com/docs/guides/auth/server-side/nextjs
 * Validates: code param present + next is same-origin (CSRF / open-redirect defense).
 */
export async function GET(request: NextRequest) {
  const url = new URL(request.url);
  const code = url.searchParams.get('code');
  const nextRaw = url.searchParams.get('next') ?? '/ko/';

  // Open-redirect guard: only allow relative paths starting with /
  const next = nextRaw.startsWith('/') && !nextRaw.startsWith('//') ? nextRaw : '/ko/';

  if (!code) {
    return NextResponse.redirect(new URL('/ko/login?error=missing_code', url.origin));
  }

  const supabase = await createClient();
  const { error } = await supabase.auth.exchangeCodeForSession(code);
  if (error) {
    // Log to Sentry when Plan 07 wires it; for now redirect with error query string.
    return NextResponse.redirect(new URL(`/ko/login?error=${encodeURIComponent(error.message)}`, url.origin));
  }

  return NextResponse.redirect(new URL(next, url.origin));
}
```

2. `src/app/auth/signout/route.ts` — POST-only signout handler. Clears session cookie + redirects to `/ko/`:
```ts
import { NextRequest, NextResponse } from 'next/server';
import { createClient } from '@/lib/supabase/server';

/**
 * POST /auth/signout — clears the session cookie.
 * POST-only prevents CSRF-triggered signouts from <img> or prefetch.
 */
export async function POST(request: NextRequest) {
  const supabase = await createClient();
  await supabase.auth.signOut();
  return NextResponse.redirect(new URL('/ko/', request.url), { status: 303 });
}
```

'use client';
import { useTranslations } from 'next-intl';
import { createClient } from '@/lib/supabase/client';

type Props = {
  disabled: boolean;
  nextPath: string;
};

const KAKAO_ENABLED = process.env.NEXT_PUBLIC_KAKAO_ENABLED === 'true';

export function KakaoButton({ disabled, nextPath }: Props) {
  const t = useTranslations('login');
  const supabase = createClient();

  async function handleClick() {
    if (!KAKAO_ENABLED) return;
    const redirectTo = `${window.location.origin}/auth/callback?next=${encodeURIComponent(nextPath)}`;
    const { error } = await supabase.auth.signInWithOAuth({
      provider: 'kakao',
      options: { redirectTo, scopes: 'account_email profile_nickname' },
    });
    if (error) {
      alert(t('oauthError', { message: error.message }));
    }
  }

  return (
    <button
      type="button"
      disabled={disabled || !KAKAO_ENABLED}
      onClick={handleClick}
      className="w-full rounded border px-4 py-2 disabled:opacity-50"
      aria-label={t('kakao')}
      title={!KAKAO_ENABLED ? t('kakaoDisabledTooltip') : undefined}
    >
      {t('kakao')}
      {!KAKAO_ENABLED && <span className="ml-2 text-xs">({t('kakaoPending')})</span>}
    </button>
  );
}

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OAuth callback | Attacker may forge `code` or hijack via intercepted redirect. |
| `/auth/callback?next=` | Open-redirect risk if `next` allowed external URL. |
| OAuth provider redirect URIs | Must be allowlisted at Google/Kakao to prevent token theft. |
| Session cookie | httpOnly, SameSite=Lax, Secure in prod (set by @supabase/ssr defaults). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-04-01 | Tampering | `/auth/callback?next=//evil.com` open redirect | mitigate | Task 1 validates `next.startsWith('/')` AND `!next.startsWith('//')` — only same-origin relative paths accepted. Same check in `/ko/login` page. |
| T-01-04-02 | Spoofing | Attacker crafts OAuth code replay | mitigate | Supabase `exchangeCodeForSession` is single-use; code binds to PKCE verifier stored in cookie. Handled by @supabase/ssr. |
| T-01-04-03 | Information Disclosure | Kakao approval before Business app grants email scope | mitigate | Feature flag `NEXT_PUBLIC_KAKAO_ENABLED` defaults false; button disabled + tooltip. `docs/auth/KAKAO_BUSINESS_APP.md` gates the flag flip. |
| T-01-04-04 | Elevation of Privilege | Fake `custom_access_token_hook` injection from malicious DB migration | accept | Migrations committed via CI + reviewed. Supabase Dashboard Hook enable is manual one-click. |
| T-01-04-05 | CSRF | Attacker triggers signout via `<img src="/auth/signout">` | mitigate | Signout route handler is POST-only — GET returns 405. Form requires same-origin submission. SameSite=Lax cookie blocks cross-site POSTs. |
| T-01-04-06 | Information Disclosure | OAuth error messages leak Supabase internals | mitigate | Errors are `encodeURIComponent`-wrapped; Sentry scrubs before logging (Plan 07). |
| T-01-04-07 | Spoofing | Missing PIPA consent bypasses user consent requirement | mitigate | `SignInPanel` disables OAuth buttons until consent checkbox checked. Server-side: PIPA compliance is about collection; OAuth buttons are the only way to create `auth.users` row and they require consent client-side. |
| T-01-04-08 | Tampering | `getSessionUser` trusts cookie without validation | mitigate | `supabase.auth.getUser()` validates JWT signature on every call. React `cache()` dedupes within a single request. |
| T-01-04-09 | Information Disclosure | `user.email` shown in user-menu without escape | accept | React auto-escapes text nodes; email is already pattern-constrained. |
| T-01-04-10 | Denial of Service | Login page requires JS for OAuth button → JS-disabled users blocked | accept | v1 persona is researchers on modern browsers; progressive enhancement deferred to v2. Signout form works without JS. |
| T-01-04-11 | Spoofing | Kakao `account_email` scope not granted (personal app) → user has no email | mitigate | Only enable Kakao after Business app approval (documented blocker). Supabase will reject signup if email missing and email is required. |
| T-01-04-12 | Information Disclosure | Session cookie readable by JS → XSS exfiltration | mitigate | @supabase/ssr sets session cookie as httpOnly by default. Tailwind + React reduce XSS risk; CSP headers in Plan 07. |
